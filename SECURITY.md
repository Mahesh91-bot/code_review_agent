# Security Policy

## Supported Versions

SAGE is actively maintained on the default branch (`main`).
Security fixes are prioritized for the latest codebase used in demos and public submissions.

## Reporting a Vulnerability

Please **do not** open public GitHub issues for security vulnerabilities.

Instead, report privately via:

- 📧 **security@sage-project.dev**

Include as much detail as possible:

- Vulnerability description
- Impacted component(s)
- Reproduction steps / proof of concept
- Suggested mitigation (if known)

## What to Expect

After receiving a report, maintainers will:

1. Acknowledge receipt within **72 hours**
2. Validate and triage severity
3. Work on a fix and coordinate disclosure timing
4. Credit reporters (if desired) after patch release

## Disclosure Guidelines

- Please allow us reasonable time to investigate and patch.
- Avoid publishing exploit details until a fix is available.
- We will coordinate responsible disclosure with reporters.

Thank you for helping keep SAGE and its community safe. 🔒
